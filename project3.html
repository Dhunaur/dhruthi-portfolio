Vehicle Pollution Control — Java desktop system

Project overview

A Java Swing desktop application for monitoring and controlling vehicle emissions. Officers can register vehicles, log CO, HC, and smoke readings, auto-flag vehicles that fail pollution norms, schedule re-tests, and generate visual reports — all backed by a MySQL database with a full admin settings panel.

Project type Java Desktop App
Year 2026
My role Researcher & Developer
Project link github.com/Dhunaur/Vehicle-Emission-Monitoring-System
  • Built a multi-panel Swing UI using CardLayout — Dashboard, Register Vehicle, Emission Check, Reports, Re-Test, and Admin Settings — navigated via a custom sidebar with role-based access for admins and users.
  • Implemented the emission check engine: reads CO, HC, and smoke levels, compares against configurable thresholds from AdminSettings, and auto-marks vehicles as Pass or Fail with a re-test cycle of 6 months.
  • Connected all panels to MySQL via a DAO layer (EmissionDAO, VehicleDAO, UserDAO, AdminSettingsDAO) with a centralized DBConnection class — keeping database logic cleanly separated from the UI.
  • Added a visual report dialog with charts showing emission trends per vehicle, plus an email alert utility (EmailUtil) to notify owners when their vehicle fails inspection.
